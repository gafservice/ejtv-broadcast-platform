# Objetivo

Establecer la configuración oficial de seguridad de red del servidor, definiendo la política del firewall, los servicios autorizados y los criterios mínimos de protección que deberán mantenerse durante la operación de la plataforma.

---

# Firewall

La plataforma utiliza **Uncomplicated Firewall (UFW)** como mecanismo oficial para la administración del firewall del sistema.

UFW administra las reglas de filtrado sobre Netfilter, constituyendo la única herramienta autorizada para modificar la configuración del firewall.

No se permite la administración manual de reglas mediante iptables.

---

# Política de seguridad

Las políticas oficiales del firewall son las siguientes:

| Dirección | Política |
| --------- | -------- |
| Entrada   | DENY     |
| Salida    | ALLOW    |
| Reenvío   | DENY     |

Esta configuración implementa una estrategia **Default Deny**, permitiendo únicamente el acceso a los servicios expresamente autorizados.

---

# Servicios autorizados

Los únicos servicios expuestos a la red son los siguientes:

| Servicio | Puerto | Protocolo | Estado     |
| -------- | -----: | --------- | ---------- |
| SSH      |     22 | TCP       | Producción |
| Cockpit  |   9090 | TCP       | Producción |

No deberán habilitarse servicios adicionales sin la correspondiente validación técnica y actualización de esta línea base.

---

# Servicios internos

Los siguientes servicios permanecen restringidos a la interfaz local del sistema y no requieren reglas de acceso externo.

| Servicio                     |   Puerto |
| ---------------------------- | -------: |
| systemd-resolved             |       53 |
| CUPS                         |      631 |
| Puertos efímeros del sistema | Variable |

---

# Administración

La administración remota del servidor se realiza mediante:

* OpenSSH.
* Cockpit.

Ambos servicios forman parte de la configuración oficial del servidor.

---

# Criterios de cumplimiento

La configuración será considerada conforme cuando se verifique que:

* UFW permanece habilitado.
* La política de entrada es **DENY**.
* La política de salida es **ALLOW**.
* La política de reenvío es **DENY**.
* Únicamente los puertos 22/TCP y 9090/TCP permanecen accesibles desde la red.
* No existen reglas manuales de iptables ajenas a UFW.

---

# Validación

La conformidad de esta línea base puede verificarse mediante los siguientes comandos:

```bash
sudo ufw status numbered verbose

sudo iptables -L -n

ss -tln

ss -uln
```
