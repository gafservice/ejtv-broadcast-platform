# Resumen Ejecutivo

Durante la ejecución de la **MISIÓN 004**, correspondiente a la actualización
del sistema operativo Ubuntu, el proceso de actualización finalizó con una
incidencia relacionada con el paquete **broadcom-sta-dkms**.

El gestor de paquetes **dpkg** no logró completar correctamente la
configuración del controlador Broadcom debido a un error durante la compilación
del módulo DKMS para el kernel **6.17.0-35-generic**.

Aunque el incidente no afecta la estabilidad general del servidor ni la
conectividad de la plataforma, impide que el gestor de paquetes alcance un
estado completamente consistente.

La presente incidencia documenta el proceso de investigación, el diagnóstico
técnico y las acciones previstas para su resolución.

# 2. Descripción del problema

La incidencia fue detectada durante la ejecución del comando:

```bash
sudo apt full-upgrade -y
```

La actualización del sistema operativo avanzó normalmente hasta la fase de
configuración de paquetes.

Durante dicha etapa el paquete **broadcom-sta-dkms** intentó compilar un módulo
del kernel correspondiente al controlador inalámbrico Broadcom instalado en el
servidor.

La compilación no pudo completarse correctamente, provocando que **dpkg**
finalizara con un código de error y dejando el paquete en estado pendiente de
configuración.

---

# 3. Evidencia experimental

## 3.1 Registro de la actualización

Toda la salida del proceso fue almacenada en el archivo:

```text
logs/mission_004_upgrade.log
```

La evidencia principal corresponde al siguiente mensaje.

```text
Setting up broadcom-sta-dkms ...

Building for 6.17.0-35-generic

Building initial module for 6.17.0-35-generic

Error! Bad return status for module build on kernel:
6.17.0-35-generic

dpkg returned error code (1)
```

---

## 3.2 Estado del paquete

La consulta realizada mediante:

```bash
apt policy broadcom-sta-dkms
```

confirmó que el paquete instalado corresponde a la versión:

```text
6.30.223.271-23ubuntu1.1
```

---

## 3.3 Estado de DKMS

La verificación mediante:

```bash
dkms status
```

mostró:

```text
broadcom-sta/6.30.223.271: added
```

Este resultado confirma que los archivos fueron registrados por DKMS, pero el
módulo nunca llegó a compilarse correctamente para el kernel instalado.

---

## 3.4 Inventario del hardware

La inspección realizada mediante:

```bash
lspci
```

identificó la presencia del siguiente dispositivo inalámbrico.

```text
Broadcom BCM4322
```

Asimismo se identificaron las interfaces Ethernet Intel utilizadas por la
plataforma.

---

## 3.5 Estado de la infraestructura de red

La verificación mediante:

```bash
nmcli device status
```

confirmó que el servidor opera exclusivamente mediante interfaces Ethernet.

No se detectaron interfaces inalámbricas activas durante la operación normal
del sistema.

---

# 4. Diagnóstico

Con base en la evidencia recopilada se concluye que:

* El hardware Broadcom BCM4322 está físicamente instalado en el servidor.
* El servidor no utiliza la interfaz inalámbrica para su operación.
* Toda la conectividad de **EJTV Broadcast Platform** se realiza mediante
  interfaces Ethernet.
* El módulo DKMS correspondiente al controlador Broadcom no logra compilarse
  para el kernel **6.17.0-35-generic**.
* El problema afecta únicamente al gestor de paquetes y no compromete la
  estabilidad general del servidor.

En consecuencia, la incidencia corresponde a un problema de compatibilidad del
controlador Broadcom con el kernel instalado y no a una falla del sistema
operativo.

# 6. Alternativas evaluadas

## Alternativa A

Reparar la compilación del módulo DKMS.

### Ventajas

* Mantener disponible la interfaz inalámbrica.
* Preservar el hardware instalado.

### Desventajas

* Requiere investigar la compatibilidad del controlador con el kernel actual.
* Incrementa el tiempo de mantenimiento.

---

## Alternativa B

Eliminar el controlador Broadcom.

### Ventajas

* Elimina la incidencia.
* Simplifica el sistema.
* Evita futuras recompilaciones DKMS.

### Desventajas

* La interfaz inalámbrica dejaría de estar disponible mientras el controlador
  permanezca desinstalado.

---

## Alternativa C

Mantener temporalmente el controlador instalado.

### Ventajas

* No modifica el estado actual del sistema.
* Permite continuar investigando la causa del problema.

### Desventajas

* El gestor de paquetes permanece en estado inconsistente.
* La MISIÓN 004 no puede darse por concluida.

---

# 7. Decisión

Al momento de elaborar este documento no se ha tomado una decisión definitiva.

Antes de modificar el sistema se documentará formalmente la decisión mediante
el correspondiente **Architecture Decision Record (ADR)** y se desarrollará un
procedimiento técnico que describa tanto la aplicación como la posible
reversión del cambio.

# Conclusión

La incidencia **INC-001** constituye el primer incidente técnico documentado
durante el desarrollo de **EJTV Broadcast Platform**.

Más allá del problema específico detectado, este documento establece el modelo
de gestión de incidencias que será utilizado durante toda la evolución del
proyecto.

Cada incidencia futura será analizada mediante evidencia experimental,
diagnóstico técnico, evaluación de alternativas y documentación de la decisión
adoptada, garantizando la trazabilidad completa de la infraestructura y
preservando el conocimiento generado durante su desarrollo.
