# Parte 4.2 — U-002 Verificación del procesador, memoria y plataforma de hardware

---

# Introducción

Una vez verificada la identidad general del servidor, el siguiente paso
consistió en caracterizar la plataforma física sobre la cual se ejecutará
**EJTV Broadcast Platform**.

Esta etapa tiene como propósito determinar las capacidades de procesamiento,
memoria y arquitectura del equipo, verificando que los recursos disponibles sean
compatibles con los requerimientos definidos durante el diseño de la
infraestructura.

A diferencia de una estación de trabajo convencional, un servidor destinado a
la distribución de contenido audiovisual debe ser capaz de mantener una
operación continua durante largos períodos de tiempo, ejecutar múltiples
servicios simultáneamente y responder de manera estable ante incrementos de la
carga de trabajo.

Por esta razón se realizó una inspección detallada del procesador, la memoria
principal y de la información suministrada por el firmware SMBIOS, obteniendo
una descripción precisa de la plataforma física utilizada durante el desarrollo
del proyecto.

---

# Objetivo

Verificar las características del procesador, la memoria principal y la
plataforma física del servidor, documentando los recursos disponibles para la
ejecución de los diferentes servicios que conformarán **EJTV Broadcast
Platform**.

---

# Comandos ejecutados

Para obtener esta información se utilizaron las siguientes herramientas de
diagnóstico incluidas en Ubuntu.

```bash
lscpu

free -h

lsmem

sudo dmidecode -t system

sudo dmidecode -t baseboard
```

---

# Descripción de los comandos utilizados

## lscpu

Obtiene información detallada del procesador reconocida por el kernel de Linux.

Entre los parámetros reportados se encuentran:

- arquitectura;
- fabricante;
- modelo;
- frecuencia;
- número de núcleos;
- número de hilos;
- memoria caché;
- capacidades de virtualización;
- mitigaciones de seguridad implementadas por el sistema operativo.

Este comando constituye una de las herramientas principales para caracterizar la
capacidad de procesamiento de un servidor.

---

## free -h

Presenta un resumen del uso actual de la memoria RAM y del espacio de
intercambio (Swap).

La opción **-h** muestra los resultados utilizando unidades de tamaño legibles
para el usuario, como MiB y GiB.

Este comando permite evaluar el consumo inicial de memoria inmediatamente
después de la instalación del sistema operativo.

---

## lsmem

Muestra la organización física de la memoria principal detectada por el kernel.

Permite conocer:

- cantidad total instalada;
- bloques de memoria;
- memoria disponible;
- memoria fuera de servicio.

Esta información resulta útil para futuras ampliaciones del servidor.

---

## dmidecode -t system

Consulta la información SMBIOS correspondiente al sistema completo.

Los datos obtenidos provienen directamente del firmware del equipo e incluyen:

- fabricante;
- modelo;
- número de serie;
- identificador único (UUID);
- familia del sistema.

Esta información resulta especialmente importante para documentar el inventario
del hardware.

---

## dmidecode -t baseboard

Consulta la información correspondiente a la placa principal del servidor.

Entre otros parámetros permite identificar:

- fabricante;
- modelo;
- número de serie;
- dispositivos integrados.

Esta información facilita futuras tareas de mantenimiento y actualización del
hardware.

# Interpretación técnica

El servidor utiliza un procesador **Intel Xeon W3530**, perteneciente a una
familia de procesadores diseñada originalmente para estaciones de trabajo y
servidores profesionales.

Aunque se trata de una plataforma con varios años de antigüedad, continúa
ofreciendo un nivel de desempeño adecuado para aplicaciones relacionadas con
administración de servicios, distribución de contenido audiovisual y ejecución
simultánea de múltiples procesos.

La presencia de **cuatro núcleos físicos y ocho hilos de ejecución** permite
distribuir diferentes tareas entre varios procesos concurrentes.

Durante el desarrollo de EJTV Broadcast Platform esta capacidad permitirá
ejecutar simultáneamente servicios como:

- MediaMTX;
- Cockpit;
- servidor SSH;
- servicios de monitoreo;
- herramientas de administración;
- procesos auxiliares del sistema operativo.

El soporte para **Intel VT-x** representa una característica importante de la
plataforma.

Aunque inicialmente no se contempla el uso de máquinas virtuales, esta
capacidad permitirá incorporar posteriormente entornos de pruebas, laboratorios
aislados o servicios virtualizados sin requerir cambios en el hardware.

En relación con la memoria principal, Ubuntu reporta una capacidad instalada de
**8 GiB**, de los cuales aproximadamente **3.8 GiB** se encontraban ocupados
durante la inspección inicial del servidor.

Debe considerarse que este valor corresponde al sistema operativo recién
instalado, incluyendo el entorno gráfico de Ubuntu Desktop y los procesos
básicos del sistema.

La memoria disponible, cercana a **4 GiB**, proporciona un margen suficiente
para comenzar la instalación de los servicios previstos durante la primera etapa
del proyecto.

No obstante, conforme aumente la cantidad de aplicaciones ejecutándose
simultáneamente, será recomendable ampliar la memoria principal para mantener
niveles adecuados de desempeño.

La información obtenida mediante SMBIOS confirma nuevamente que el sistema
operativo reconoce correctamente el hardware del servidor.

El fabricante identificado corresponde a **Apple Inc.**, mientras que el modelo
reportado es **MacPro5,1**, coincidiendo con la plataforma seleccionada durante
la fase de diseño.

Esta correcta identificación demuestra la elevada compatibilidad existente entre
Ubuntu y el hardware instalado.

---

# Importancia para el proyecto

La caracterización del hardware permite establecer la línea base de capacidad de
procesamiento sobre la cual evolucionará **EJTV Broadcast Platform**.

A partir de esta información será posible evaluar el impacto que producirán la
instalación de nuevos servicios, el incremento del tráfico de red y el
crecimiento de la infraestructura.

Asimismo, estos datos servirán como referencia para futuras comparaciones cuando
se incorporen ampliaciones de memoria, nuevos procesadores o mecanismos de
virtualización.

La disponibilidad de esta información desde las primeras etapas del proyecto
facilita la planificación del crecimiento de la plataforma y reduce la
incertidumbre durante futuras actualizaciones del hardware.

---

# Conclusión

La evidencia recopilada demuestra que el servidor dispone de una plataforma de
hardware sólida y adecuada para la primera etapa de desarrollo de
**EJTV Broadcast Platform**.

El procesador Intel Xeon W3530, la arquitectura de 64 bits, la disponibilidad
de ocho hilos de procesamiento y los 8 GiB de memoria principal proporcionan
una base estable para ejecutar simultáneamente los servicios previstos durante
las primeras fases del proyecto.

La correcta identificación del hardware por parte de Ubuntu confirma además la
compatibilidad entre el sistema operativo y la plataforma Apple Mac Pro 5,1,
permitiendo continuar con confianza hacia las siguientes etapas de
configuración, almacenamiento y despliegue de servicios.
