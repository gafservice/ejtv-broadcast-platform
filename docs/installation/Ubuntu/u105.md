# Parte 4.5 — U-005 Verificación del estado operativo del sistema

---

# Introducción

Una vez concluida la instalación del sistema operativo y verificados los
componentes físicos del servidor, el siguiente paso consistió en evaluar el
estado operativo general de la plataforma.

El objetivo de esta inspección no fue únicamente comprobar que Ubuntu iniciara
correctamente.

También se buscó verificar que los principales servicios del sistema se
encontraran funcionando normalmente, que el usuario administrativo poseyera los
privilegios adecuados, que la sincronización horaria estuviera correctamente
configurada y que no existieran errores que pudieran comprometer las etapas
posteriores del proyecto.

Esta revisión representa la primera evaluación funcional del servidor
**ejtv-01** después de su instalación.

La información obtenida constituye la línea base del estado operativo del
sistema y permitirá detectar fácilmente cualquier anomalía que pueda aparecer
durante futuras modificaciones de la plataforma.

---

# Objetivo

Verificar el estado operativo general del servidor inmediatamente después de la
instalación de Ubuntu, confirmando el funcionamiento de los servicios
esenciales, la configuración del usuario administrativo y la estabilidad del
sistema.

---

# Comandos ejecutados

Para realizar esta inspección se utilizaron los siguientes comandos.

```bash
timedatectl

uptime

whoami

groups

id

systemctl --failed

systemctl status ssh --no-pager

systemctl status NetworkManager --no-pager
```

---

# Descripción de los comandos utilizados

## timedatectl

Consulta la configuración relacionada con el reloj del sistema.

Permite verificar:

- fecha y hora actuales;
- zona horaria;
- sincronización mediante NTP;
- estado del reloj del sistema.

Una configuración horaria correcta resulta indispensable para mantener la
coherencia de los registros del sistema, la autenticación y los procesos de
monitoreo.

---

## uptime

Presenta el tiempo transcurrido desde el último arranque del servidor.

Además informa:

- carga promedio del sistema;
- número de usuarios conectados;
- tiempo continuo de operación.

Este comando constituye uno de los principales indicadores de estabilidad del
servidor.

---

## whoami

Indica el usuario actualmente autenticado en la sesión.

Su utilización permite confirmar que las tareas administrativas están siendo
realizadas con el usuario previsto durante el diseño de la plataforma.

---

## groups

Presenta los grupos de seguridad a los cuales pertenece el usuario actual.

La pertenencia a determinados grupos determina los privilegios disponibles para
administrar el sistema.

---

## id

Muestra los identificadores numéricos del usuario y de sus grupos.

Esta información permite verificar la correcta asignación de privilegios
administrativos.

---

## systemctl --failed

Consulta todos los servicios administrados por **systemd** que presenten fallos
durante el arranque del sistema.

La ausencia de unidades fallidas constituye uno de los principales indicadores
de una instalación exitosa.

---

## systemctl status ssh

Verifica el estado del servicio **OpenSSH Server**.

Este servicio permitirá administrar remotamente el servidor durante las
siguientes etapas del proyecto.

---

## systemctl status NetworkManager

Consulta el estado del servicio responsable de administrar la conectividad de
red del servidor.

Su correcto funcionamiento resulta indispensable para mantener la comunicación
con el resto de la infraestructura.

# Análisis de la evidencia

La información obtenida demuestra que el sistema operativo inició correctamente
y que los principales servicios esenciales se encuentran funcionando de manera
normal.

El usuario autenticado corresponde a **ejtv**, creado durante la instalación del
sistema y configurado como administrador principal del servidor.

Este usuario será utilizado durante todas las etapas iniciales del proyecto,
permitiendo mantener una trazabilidad clara de las actividades realizadas sobre
la plataforma.

La sincronización automática mediante NTP quedó correctamente habilitada.

Este aspecto adquiere especial importancia dentro de una infraestructura de
servidores, ya que garantiza la coherencia temporal de registros, bitácoras,
certificados digitales y mecanismos de autenticación.

La revisión de los servicios administrados por **systemd** no reportó unidades
fallidas.

Este resultado constituye uno de los indicadores más importantes de una
instalación estable, ya que confirma que todos los servicios esenciales
completaron correctamente su proceso de inicialización.

Asimismo, tanto el servicio **SSH** como **NetworkManager** se encontraban en
estado **active (running)** durante la inspección.

Esto confirma que el servidor puede ser administrado tanto localmente como de
forma remota y que dispone de conectividad de red completamente funcional.

---

# Implicaciones para el proyecto

La correcta operación de los servicios básicos constituye el requisito previo
para continuar con la instalación del resto de la infraestructura.

A partir de este punto será posible incorporar progresivamente nuevos servicios
como:

- Git;
- Visual Studio Code;
- Cockpit;
- MediaMTX;
- Fail2Ban;
- herramientas de monitoreo;
- servicios de distribución audiovisual.

El estado estable observado durante esta inspección reduce considerablemente el
riesgo de que futuras incidencias se deban a problemas relacionados con la
instalación inicial del sistema operativo.

Además, esta evidencia constituye la referencia oficial del estado operativo del
servidor antes de comenzar cualquier modificación adicional de la plataforma.

---

# Conclusión

La auditoría del estado operativo confirma que **ejtv-01** se encuentra
funcionando de manera estable y que la instalación inicial de Ubuntu fue
realizada correctamente.

El usuario administrativo dispone de los privilegios necesarios para continuar
con la configuración del sistema, la sincronización horaria opera
adecuadamente, los servicios esenciales permanecen activos y no se detectaron
fallos durante el arranque del sistema.

Esta condición permite considerar concluida la etapa de instalación del sistema
operativo y proporciona una base sólida para iniciar la incorporación de los
componentes que conformarán **EJTV Broadcast Platform**.
