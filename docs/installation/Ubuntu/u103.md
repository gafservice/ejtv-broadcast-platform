# Parte 4.3 — U-003 Verificación del subsistema de almacenamiento

---

# Introducción

El subsistema de almacenamiento constituye uno de los componentes más críticos
de cualquier servidor.

Sobre él no solamente reside el sistema operativo, sino también la información
generada durante la operación diaria, los registros del sistema, los archivos de
configuración, las grabaciones, las copias de seguridad y, en el caso de
**EJTV Broadcast Platform**, el contenido audiovisual distribuido hacia los
clientes.

Por esta razón, una vez finalizada la instalación de Ubuntu, se realizó una
caracterización completa del almacenamiento disponible en el servidor con el
objetivo de verificar la correcta instalación del sistema operativo, identificar
las unidades físicas presentes y documentar el estado inicial de cada una de
ellas.

Esta inspección constituye la línea base del almacenamiento y servirá como
referencia para todas las futuras ampliaciones de la plataforma.

---

# Objetivo

Inventariar las unidades de almacenamiento instaladas en el servidor,
identificar la ubicación del sistema operativo, verificar el esquema de
particiones y documentar el estado inicial del almacenamiento disponible para el
proyecto.

---

# Comandos ejecutados

Para realizar esta inspección se utilizaron las siguientes herramientas de
diagnóstico incluidas en Ubuntu.

```bash
lsblk -o NAME,SIZE,FSTYPE,MOUNTPOINT,MODEL

sudo fdisk -l

df -h

blkid
```

---

# Descripción de los comandos utilizados

## lsblk

Presenta una representación jerárquica de todos los dispositivos de bloques
detectados por el sistema operativo.

Permite identificar:

- discos físicos;
- particiones;
- sistemas de archivos;
- puntos de montaje;
- modelo de cada unidad.

Es la herramienta principal para obtener una visión general del almacenamiento.

---

## fdisk -l

Muestra información detallada de las tablas de particiones presentes en cada
disco.

Entre otros aspectos permite conocer:

- tipo de tabla de particiones;
- tamaño del disco;
- tipo de cada partición;
- sectores utilizados;
- identificadores GPT.

Esta información resulta especialmente útil para documentar la estructura física
del almacenamiento.

---

## df -h

Presenta el uso actual del espacio disponible en los sistemas de archivos
montados.

La opción **-h** muestra los resultados utilizando unidades legibles para el
usuario.

Este comando permite verificar el nivel de ocupación del sistema operativo una
vez concluida la instalación.

---

## blkid

Obtiene el identificador único (UUID) de cada sistema de archivos reconocido por
el kernel.

Los UUID son utilizados por Linux para montar automáticamente las particiones,
independientemente del orden en que sean detectados los discos durante el
arranque.

# Análisis de la evidencia

La información recopilada confirma que la instalación de Ubuntu se realizó
exclusivamente sobre la unidad SSD Samsung de 512 GB.

Esta decisión permitió mantener completamente aislado el sistema operativo del
resto de las unidades presentes en el servidor.

La utilización de un SSD para alojar el sistema operativo proporciona tiempos
de arranque reducidos, mayor velocidad durante la carga de aplicaciones y una
respuesta considerablemente superior frente a discos mecánicos tradicionales.

Otro aspecto relevante corresponde a la preservación de los tres discos
mecánicos instalados originalmente en el servidor.

Ubuntu detectó correctamente estas unidades, identificando sus particiones HFS+
sin modificar su contenido.

Esta decisión evita la pérdida de información previa y mantiene disponibles
estas unidades para futuras tareas de migración o reutilización.

El nivel de ocupación del sistema operativo, cercano al seis por ciento de la
capacidad disponible, indica que la instalación inicial presenta un amplio
margen para incorporar nuevas aplicaciones, servicios y herramientas sin riesgo
de saturar el almacenamiento principal.

La utilización del sistema de archivos **EXT4** proporciona una plataforma
estable, ampliamente soportada dentro del ecosistema Linux y adecuada para un
servidor de propósito general.

Finalmente, la existencia de identificadores UUID garantiza que las particiones
podrán montarse correctamente incluso si cambia el orden de detección de los
discos durante futuros reinicios del sistema.

---

# Implicaciones para el proyecto

La organización actual del almacenamiento coincide con la arquitectura definida
durante las primeras etapas de diseño de EJTV Broadcast Platform.

La separación física entre el sistema operativo y las unidades de datos ofrece
varias ventajas importantes.

En primer lugar, facilita las tareas de mantenimiento y reinstalación del
sistema operativo sin afectar el contenido almacenado en los discos destinados a
producción.

En segundo lugar, permite asignar funciones específicas a cada unidad conforme
la plataforma evolucione.

Entre las aplicaciones previstas para estos discos se consideran:

- almacenamiento de contenido audiovisual;
- grabaciones de transmisiones;
- copias de seguridad;
- archivos de configuración históricos;
- repositorios multimedia;
- almacenamiento temporal de procesos de ingest.

Esta estrategia simplifica la administración del servidor y proporciona una base
adecuada para futuras ampliaciones del almacenamiento.

# Conclusión

La auditoría del subsistema de almacenamiento confirma que el servidor dispone
de una infraestructura sólida y adecuadamente organizada para continuar con el
desarrollo de **EJTV Broadcast Platform**.

La instalación de Ubuntu quedó confinada a una unidad SSD dedicada, mientras que
los tres discos mecánicos originales permanecieron intactos, preservando tanto
su estructura como la información previamente almacenada.

La disponibilidad de aproximadamente **419 GB libres** en la unidad principal y
de más de **4 TB** adicionales distribuidos entre las restantes unidades
proporciona una capacidad suficiente para soportar las siguientes etapas del
proyecto y el crecimiento previsto de la plataforma durante los próximos años.
