# Parte 4.9 — U-009 Verificación del kernel y del registro de eventos del sistema

---

# Introducción

Una vez establecida la línea base del servidor y documentadas las principales
características del hardware y del sistema operativo, el siguiente paso
consistió en verificar el estado del kernel de Linux y analizar los registros
generados durante el proceso de arranque.

El kernel constituye el componente central del sistema operativo.

Es el responsable de administrar el procesador, la memoria, los dispositivos de
almacenamiento, las interfaces de red y todos los recursos físicos del
servidor.

Durante el proceso de inicialización, el kernel registra continuamente mensajes
informativos, advertencias y errores relacionados con el hardware y los
controladores cargados.

El análisis de estos registros permite detectar tempranamente problemas de
compatibilidad, fallos de dispositivos o configuraciones incorrectas antes de
incorporar nuevos servicios al servidor.

Por esta razón, la verificación del kernel forma parte de la auditoría inicial
de **ejtv-01**.

---

# Objetivo

Verificar el estado operativo del kernel de Linux y documentar los mensajes
generados durante el primer arranque del servidor, estableciendo una referencia
para futuras comparaciones.

---

# Comandos ejecutados

Para realizar esta inspección se utilizaron las siguientes herramientas.

```bash
uname -r

lsmod

dmesg --level=err,warn

journalctl -b -p warning
```

---

# Descripción de los comandos utilizados

## uname -r

Muestra la versión exacta del kernel actualmente en ejecución.

Esta información resulta indispensable para documentar futuras actualizaciones
del sistema operativo.

---

## lsmod

Presenta la lista de módulos actualmente cargados por el kernel.

Cada módulo corresponde a un controlador responsable de administrar un
determinado dispositivo físico.

El análisis de esta información permite confirmar que Ubuntu reconoció
correctamente el hardware instalado.

---

## dmesg --level=err,warn

Consulta únicamente los mensajes del kernel clasificados como advertencias o
errores.

De esta manera es posible concentrar el análisis en aquellos eventos que
requieren atención.

---

## journalctl -b -p warning

Consulta el registro del sistema correspondiente al último arranque,
mostrando únicamente eventos con prioridad de advertencia o superior.

Esta herramienta complementa la información obtenida mediante **dmesg** y
permite verificar el estado general del proceso de inicialización.

# Análisis de la evidencia

El análisis del kernel permitirá confirmar que Ubuntu reconoce correctamente el
hardware del servidor y que los controladores necesarios fueron cargados sin
inconvenientes.

Asimismo, la revisión de los registros del sistema permitirá identificar
posibles advertencias relacionadas con dispositivos, firmware o servicios que
puedan requerir atención durante las siguientes etapas del proyecto.

La ausencia de errores críticos constituirá un indicador adicional de la
estabilidad alcanzada por el servidor inmediatamente después de su instalación.

---

# Implicaciones para el proyecto

El kernel constituye el componente fundamental sobre el cual operarán todos los
servicios de **EJTV Broadcast Platform**.

Disponer de una caracterización inicial de su estado permitirá comparar futuras
actualizaciones del sistema operativo, validar nuevos controladores y detectar
cambios producidos por la incorporación de hardware adicional.

Esta información también facilitará futuras tareas de diagnóstico cuando se
implementen servicios de distribución audiovisual y administración remota.
