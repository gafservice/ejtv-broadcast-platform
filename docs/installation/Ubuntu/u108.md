# Parte 4.8 — U-008 Caracterización del proceso de arranque del sistema

---

# Introducción

Una vez establecida la línea base del servidor, resulta conveniente documentar
el comportamiento del proceso de arranque del sistema operativo.

El tiempo requerido para iniciar un servidor constituye uno de los indicadores
más importantes de su estado operativo.

Durante la evolución de una plataforma informática es normal que el tiempo de
inicio aumente como consecuencia de la incorporación de nuevos servicios,
controladores y aplicaciones.

Por esta razón, antes de comenzar la instalación de los componentes propios de
**EJTV Broadcast Platform**, se realizó una caracterización del proceso de
arranque de Ubuntu.

La información obtenida durante esta etapa permitirá comparar el comportamiento
del servidor conforme evolucione el proyecto y facilitará la identificación de
servicios que eventualmente puedan afectar el tiempo de inicio del sistema.

---

# Objetivo

Caracterizar el proceso de arranque del servidor **ejtv-01**, documentando el
tiempo requerido para iniciar Ubuntu y los servicios que participan durante la
secuencia de inicialización.

---

# Comandos ejecutados

Para realizar esta inspección se utilizaron las herramientas de análisis
incluidas en **systemd**.

```bash
systemd-analyze

systemd-analyze blame

systemd-analyze critical-chain
```

---

# Descripción de los comandos utilizados

## systemd-analyze

Calcula el tiempo total requerido para completar el proceso de arranque del
sistema operativo.

El resultado se divide normalmente en tres etapas:

* firmware;
* cargador de arranque;
* espacio de usuario (userspace).

Esta información proporciona una visión general del desempeño del servidor
durante el inicio.

---

## systemd-analyze blame

Presenta una lista ordenada de los servicios administrados por **systemd** según
el tiempo requerido para inicializarse.

Los servicios más lentos aparecen al inicio del listado.

Esta herramienta facilita la identificación de procesos que podrían incrementar
el tiempo de arranque del servidor.

---

## systemd-analyze critical-chain

Muestra la cadena crítica de dependencias utilizada durante el proceso de
arranque.

Permite determinar cuáles servicios condicionan directamente el tiempo total de
inicio del sistema.

# Análisis de la evidencia

El análisis del proceso de arranque permitirá establecer un indicador objetivo
del desempeño inicial del servidor.

Este valor constituirá la referencia para evaluar el impacto producido por la
instalación de nuevos servicios durante el desarrollo del proyecto.

Asimismo, la identificación de los servicios con mayor tiempo de inicialización
facilitará futuras tareas de optimización y mantenimiento.

---

# Implicaciones para el proyecto

La plataforma **EJTV Broadcast Platform** incorporará progresivamente nuevos
componentes de software, entre ellos servicios de administración, monitoreo,
seguridad y distribución audiovisual.

Cada uno de estos componentes incrementará, en mayor o menor medida, el tiempo
de inicio del servidor.

La caracterización realizada en esta etapa permitirá cuantificar dicho impacto y
mantener una referencia objetiva del comportamiento del sistema desde su estado
original.

---

# Conclusión

La caracterización del proceso de arranque constituye un complemento natural de
la línea base establecida en **BL-001**.

La información obtenida permitirá documentar la evolución del servidor durante
todo el ciclo de vida del proyecto y facilitará la detección temprana de
servicios que puedan afectar el desempeño general del sistema.

Una vez incorporada la evidencia experimental, esta sección representará el
registro oficial del comportamiento del proceso de arranque de **ejtv-01** antes
de iniciar la instalación de los componentes propios de **EJTV Broadcast
Platform**.
