# Parte 4.4 — U-004 Verificación de la infraestructura de red

---

# Introducción

Una vez verificados el sistema operativo, la plataforma de hardware y el
subsistema de almacenamiento, el siguiente paso consistió en inspeccionar la
infraestructura de red del servidor.

La comunicación constituye el componente fundamental de
**EJTV Broadcast Platform**, ya que todos los servicios previstos para la
plataforma dependen del correcto funcionamiento de la red.

Servicios como SSH, Cockpit, MediaMTX, NTP, Git, los futuros mecanismos de
monitoreo y la distribución de contenido audiovisual requieren una
infraestructura de comunicaciones estable y correctamente configurada.

Por esta razón, antes de instalar cualquier servicio adicional, se realizó una
caracterización completa de las interfaces de red presentes en el servidor.

Esta inspección permitió verificar que Ubuntu reconociera correctamente las
tarjetas Ethernet instaladas, identificar la interfaz utilizada durante la
instalación y documentar la configuración inicial de la conectividad.

La información obtenida constituye la línea base sobre la cual se desarrollará
posteriormente toda la arquitectura de comunicaciones del proyecto.

---

# Objetivo

Verificar la infraestructura de red instalada en el servidor, documentar las
interfaces Ethernet disponibles y confirmar el correcto funcionamiento de la
configuración inicial de red.

---

# Comandos ejecutados

Para realizar esta inspección se utilizaron las siguientes herramientas de
diagnóstico.

```bash
ip addr

ip route

ip link

nmcli device status
```

---

# Descripción de los comandos utilizados

## ip addr

Muestra todas las interfaces de red detectadas por el kernel de Linux.

Para cada interfaz presenta información relacionada con:

- nombre;
- dirección MAC;
- direcciones IPv4;
- direcciones IPv6;
- estado operativo.

Actualmente constituye la herramienta recomendada para reemplazar comandos
históricos como **ifconfig**.

---

## ip route

Presenta la tabla de enrutamiento utilizada por el sistema operativo.

Permite conocer:

- puerta de enlace predeterminada;
- redes locales;
- interfaces utilizadas para cada ruta.

Toda comunicación IP generada por el servidor depende de esta información.

---

## ip link

Muestra el estado físico de cada interfaz de red.

Entre otros parámetros informa si la interfaz se encuentra:

- activa;
- deshabilitada;
- conectada;
- desconectada.

Este comando permite diferenciar problemas físicos de problemas de
configuración.

---

## nmcli device status

Consulta el estado de los dispositivos administrados por
**NetworkManager**.

Permite conocer:

- interfaces activas;
- interfaces desconectadas;
- tipo de conexión;
- estado administrativo.

Resulta especialmente útil en servidores administrados mediante Ubuntu Desktop.

```text

```

El servicio fue detectado correctamente y se encontraba operativo durante toda

La plataforma dispone de cuatro interfaces físicas Ethernet.

| Interfaz | Tecnología |
|-----------|------------------------------|
| enp9s0 | Intel Gigabit Ethernet |
| enp10s0 | Intel Gigabit Ethernet |
| ens2f0 | Intel X540 10 Gigabit Ethernet |
| ens2f1 | Intel X540 10 Gigabit Ethernet |

Esta configuración proporciona una infraestructura de comunicaciones muy
superior a la requerida durante la primera etapa del proyecto.

# Análisis de la evidencia

Ubuntu reconoció correctamente todas las interfaces de red instaladas en el
servidor.

Durante la instalación únicamente fue necesario habilitar una de ellas para
proporcionar conectividad al sistema operativo.

La dirección IP asignada pertenece al segmento
**192.168.33.0/24**, confirmando la correcta integración del servidor dentro de
la red local utilizada durante el desarrollo del proyecto.

La tabla de enrutamiento muestra una única puerta de enlace predeterminada,
suficiente para la etapa inicial de instalación y administración.

El servicio **NetworkManager** fue detectado funcionando correctamente,
encargándose de administrar la configuración de las interfaces de red del
servidor.

La existencia de cuatro interfaces físicas constituye uno de los aspectos más
importantes observados durante esta inspección.

Dos de ellas corresponden a interfaces Gigabit Ethernet, mientras que las dos
restantes pertenecen a una tarjeta Intel X540 con capacidad de operar enlaces de
10 Gigabits por segundo.

Esta configuración ofrece una capacidad de crecimiento considerablemente mayor
a la requerida para la primera etapa del proyecto.

---

# Implicaciones para el proyecto

La infraestructura de red disponible permitirá implementar una arquitectura
modular basada en la separación física de servicios.

Aunque inicialmente únicamente se utilizará una interfaz para la administración
general del servidor, la disponibilidad de cuatro interfaces Ethernet abre la
posibilidad de asignar funciones específicas a cada una de ellas.

Entre las aplicaciones previstas se consideran:

- red administrativa;
- ingest de contenido audiovisual;
- distribución SRT;
- monitoreo;
- laboratorio de pruebas;
- futuras VLAN.

Esta estrategia facilitará el aislamiento del tráfico de red, mejorará la
seguridad de la plataforma y permitirá incrementar la capacidad del sistema sin
modificar la arquitectura física del servidor.

La disponibilidad de interfaces de 10 Gigabits representa además una ventaja
importante para futuras aplicaciones relacionadas con transporte de video de
alta resolución y almacenamiento distribuido.

---

# Conclusión

La auditoría inicial confirma que Ubuntu reconoció correctamente toda la
infraestructura de comunicaciones instalada en el servidor.

El sistema quedó conectado mediante una interfaz Gigabit Ethernet,
obteniendo conectividad plena con la red local y acceso a Internet.

Las restantes interfaces permanecen disponibles para futuras etapas del
proyecto, proporcionando una plataforma de comunicaciones altamente flexible y
preparada para soportar el crecimiento previsto de
**EJTV Broadcast Platform**.

La infraestructura de red instalada supera ampliamente los requerimientos de la
primera versión del proyecto y constituye una base sólida para la construcción
de la arquitectura de comunicaciones que será desarrollada en los siguientes
capítulos.
