# Parte 4.1 — U-001 Identificación del servidor

---

# Introducción

Una vez finalizada la instalación del sistema operativo, el primer paso
consistió en verificar la identidad general del servidor.

Aunque esta verificación puede parecer un procedimiento sencillo, representa
una de las tareas más importantes durante la puesta en marcha de cualquier
infraestructura informática.

La correcta identificación del equipo permite confirmar que el sistema
operativo fue instalado sobre la plataforma prevista, que el nombre del
servidor coincide con la arquitectura definida para el proyecto y que las
características fundamentales del hardware fueron reconocidas correctamente por
el kernel de Linux.

Además, esta información constituye el punto de partida para todas las tareas
de administración posteriores, ya que muchos servicios utilizan estos datos para
registrar eventos, generar bitácoras, establecer conexiones de red e
identificarse dentro de una infraestructura con múltiples servidores.

Por esta razón, la primera evidencia recopilada durante la auditoría técnica
consistió en verificar la identidad completa del servidor **ejtv-01**.

---

# Objetivo

Verificar la identidad del servidor, confirmar la correcta instalación del
sistema operativo y documentar las principales características generales de la
plataforma física reconocidas por Ubuntu.

---

# Comandos ejecutados

Para obtener esta información se utilizaron las herramientas estándar
proporcionadas por Ubuntu.

Cada una de ellas consulta una parte diferente del sistema operativo, por lo
que su utilización conjunta permite construir una descripción completa del
servidor.

```bash
hostnamectl

hostname

uname -a

lsb_release -a
```

---

# Descripción de los comandos utilizados

## hostnamectl

Consulta la información administrada por **systemd-hostnamed**.

Permite conocer el nombre del servidor, la versión del sistema operativo, el
modelo del hardware, el fabricante y otros datos generales de la plataforma.

Actualmente constituye el método recomendado por Ubuntu para identificar un
equipo basado en systemd.

---

## hostname

Muestra únicamente el nombre corto asignado al servidor.

Este nombre es utilizado por numerosos servicios internos y aparece en la línea
de comandos, registros del sistema y conexiones remotas mediante SSH.

---

## uname -a

Presenta información relacionada con el núcleo del sistema operativo.

Entre otros datos permite identificar:

- versión del kernel;
- arquitectura del procesador;
- fecha de compilación;
- nombre del servidor.

Esta información resulta especialmente útil para verificar compatibilidad con
controladores y dispositivos de hardware.

---

## lsb_release -a

Consulta la información correspondiente a la distribución Linux instalada.

Permite identificar:

- distribución;
- versión;
- nombre de la versión;
- identificador oficial.

Este comando resulta muy útil durante tareas de soporte y mantenimiento, ya que
muchas aplicaciones requieren conocer exactamente la versión del sistema
operativo donde serán instaladas.

---

# Interpretación técnica

La información obtenida confirma que la instalación del sistema operativo se
realizó correctamente sobre la plataforma física prevista durante el diseño de
la infraestructura.

El nombre permanente del servidor quedó configurado como **ejtv-01**,
manteniendo la convención de nombres definida para EJTV Broadcast Platform.

Esta nomenclatura facilitará la incorporación de nuevos servidores conforme la
infraestructura crezca, permitiendo utilizar identificadores como **ejtv-02**,
**ejtv-03** y así sucesivamente.

La distribución instalada corresponde a **Ubuntu 24.04.4 LTS**, versión de
soporte extendido orientada a ofrecer estabilidad y mantenimiento durante varios
años.

La selección de una versión LTS reduce la necesidad de realizar migraciones
frecuentes del sistema operativo y proporciona una plataforma adecuada para el
desarrollo continuo del proyecto.

El kernel identificado como **Linux 6.17.0-35-generic** incorpora soporte para
la mayoría del hardware moderno y mantiene compatibilidad con los dispositivos
instalados en el servidor, incluyendo las interfaces de red Intel, la tarjeta
gráfica AMD y las diferentes tarjetas PCI Express utilizadas durante el
proyecto.

Otro aspecto importante corresponde al reconocimiento automático del hardware.

Ubuntu identificó correctamente tanto el fabricante (**Apple Inc.**) como el
modelo (**MacPro5,1**) y la versión del firmware instalada.

Esta información confirma que el sistema operativo reconoce adecuadamente la
plataforma física sin requerir controladores propietarios adicionales para su
identificación básica.

Finalmente, la arquitectura reportada como **x86-64** confirma que el sistema
opera completamente en modo de 64 bits, permitiendo aprovechar la totalidad de
la memoria instalada y ejecutar aplicaciones modernas sin restricciones propias
de arquitecturas de 32 bits.

---

# Importancia para el proyecto

La correcta identificación del servidor constituye un requisito fundamental para
las siguientes etapas del proyecto.

Servicios como SSH, Cockpit, MediaMTX, Fail2Ban y los futuros componentes de
administración utilizarán esta información para generar registros, establecer
comunicaciones de red y facilitar la administración remota del servidor.

Asimismo, esta evidencia representa el primer registro oficial del estado de la
plataforma después de concluir la instalación del sistema operativo.

Cualquier modificación futura relacionada con el nombre del servidor, la versión
del sistema operativo o el kernel podrá compararse con esta línea base para
mantener la trazabilidad de la infraestructura.

---

# Conclusión

La evidencia obtenida demuestra que el servidor fue instalado correctamente y
que Ubuntu reconoce de manera adecuada tanto el hardware como la configuración
general del sistema.

El servidor **ejtv-01** quedó identificado de forma permanente dentro de la
infraestructura de EJTV Broadcast Platform, ejecutando Ubuntu 24.04.4 LTS sobre
una plataforma Apple Mac Pro 5,1 con arquitectura de 64 bits y kernel Linux
6.17.

Esta información constituye la primera referencia técnica oficial del proyecto y
servirá como punto de comparación para todas las futuras actualizaciones del
sistema operativo y del hardware del servidor.
