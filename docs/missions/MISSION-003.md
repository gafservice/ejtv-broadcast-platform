# MISIÓN 003

# Construcción del Entorno de Ingeniería

---

## Resumen Ejecutivo

La tercera misión del proyecto **EJTV Broadcast Platform** tuvo como propósito construir el entorno de ingeniería que acompañará el desarrollo de la plataforma durante todas sus etapas.

Durante esta misión se incorporaron las herramientas necesarias para el desarrollo de software, el control de versiones y la documentación técnica del proyecto. Asimismo, se definió la estructura inicial del repositorio y se establecieron los principios editoriales que regirán la elaboración de toda la documentación.

Al concluir esta etapa, la plataforma no solamente dispone de un servidor operativo, sino también de un entorno de ingeniería completamente funcional y preparado para acompañar el crecimiento del proyecto.

## Objetivos

El objetivo principal de esta misión fue establecer un entorno de desarrollo homogéneo, reproducible y completamente documentado que sirva como base para la construcción de **EJTV Broadcast Platform**.

Para alcanzar este objetivo se realizaron las siguientes actividades:

* Instalación de Visual Studio Code.
* Configuración inicial del entorno de desarrollo.
* Configuración de Git.
* Integración segura con GitHub mediante claves SSH.
* Creación de la estructura documental del repositorio.
* Definición de la filosofía y principios del proyecto.
* Organización inicial de la documentación técnica.

---

## Antecedentes

Todo proyecto de ingeniería requiere un entorno de trabajo estable y bien organizado.

Más allá del hardware y del sistema operativo, resulta indispensable disponer de herramientas que permitan desarrollar software, documentar decisiones técnicas y preservar la evolución del proyecto mediante un sistema de control de versiones.

Por esta razón, la tercera misión estuvo orientada a construir el laboratorio de ingeniería que acompañará el desarrollo de EJTV Broadcast Platform durante todo su ciclo de vida.

## Actividades realizadas

Durante esta misión se ejecutaron las siguientes actividades:

* Instalación de Visual Studio Code.
* Configuración del editor de desarrollo.
* Configuración de Git con la identidad del desarrollador.
* Generación de un par de claves SSH utilizando Ed25519.
* Registro de la clave pública en GitHub.
* Configuración del acceso seguro al repositorio mediante SSH.
* Organización inicial del repositorio.
* Definición de la estructura documental del proyecto.
* Elaboración de los primeros documentos fundacionales.

---

## Decisiones de ingeniería

Durante esta etapa se adoptaron las siguientes decisiones:

* Visual Studio Code será el entorno oficial de desarrollo.
* Git será el sistema oficial de control de versiones.
* La autenticación con GitHub se realizará exclusivamente mediante claves SSH.
* Markdown será el formato principal para la documentación técnica.
* La documentación crecerá de manera modular y organizada por áreas funcionales.
* Cada misión será documentada siguiendo una estructura homogénea.

Estas decisiones buscan garantizar la trazabilidad, reproducibilidad y mantenibilidad del proyecto durante todo su ciclo de vida.

---

## Validación

Al concluir esta misión se verificó que:

* Visual Studio Code funciona correctamente.
* Git reconoce la identidad del desarrollador.
* GitHub autentica correctamente mediante SSH.
* El repositorio puede sincronizarse sin utilizar contraseñas.
* La estructura documental quedó creada.
* Los documentos fundacionales fueron incorporados al proyecto.

---

## Lecciones aprendidas

La construcción temprana del entorno de ingeniería facilita considerablemente el desarrollo posterior del proyecto.

Asimismo, se confirmó que invertir tiempo en definir una estructura documental sólida desde las primeras etapas reduce el esfuerzo de mantenimiento y mejora la trazabilidad técnica de todas las decisiones adoptadas.

---

## Estado de la plataforma

Al concluir la tercera misión, **ejtv-01** dispone de un entorno completo para el desarrollo de software y la documentación técnica.

El proyecto cuenta con una estructura organizada, un sistema de control de versiones plenamente operativo y una metodología de trabajo definida que permitirá mantener la coherencia de toda la documentación futura.

---

## Conclusión

La tercera misión representa el nacimiento del entorno de ingeniería de **EJTV Broadcast Platform**.

A partir de esta etapa, el proyecto dispone no solamente de una infraestructura técnica, sino también de las herramientas y procedimientos necesarios para desarrollar, documentar y mantener la plataforma de forma ordenada y reproducible.

El entorno construido durante esta misión será el punto de partida para todas las actividades de desarrollo que seguirán en las próximas etapas del proyecto.
